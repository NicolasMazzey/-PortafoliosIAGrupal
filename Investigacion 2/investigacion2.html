Tratamiento previo de los datos y fundamentos de los algoritmos ML

¿Qué es el 'machine learning' y para qué sirve?

El Machine Learning es una rama de la Inteligencia Artificial que permite a las computadoras aprender de los datos mediante el uso de algoritmos.

En lugar de seguir instrucciones programadas de manera rígida, los sistemas pueden identificar patrones en grandes conjuntos de datos y hacer predicciones o tomar decisiones basadas en esos patrones. Este enfoque permite automatizar tareas complejas y realizar análisis predictivos.

Distintos algoritmos de ML

Aprendizaje supervisado:

En este tipo de algoritmos, el modelo aprende a partir de datos etiquetados, es decir, ejemplos previos que incluyen la respuesta correcta. A través de este proceso, el sistema puede hacer predicciones o tomar decisiones basadas en patrones ya conocidos.

Un ejemplo típico es un filtro de correo electrónico que identifica si un mensaje es spam o no, utilizando información como el remitente o las palabras clave.

Aprendizaje no supervisado

A diferencia del supervisado, estos algoritmos trabajan sin etiquetas o respuestas previas. El sistema analiza los datos buscando patrones ocultos que puedan agrupar o categorizar la información.

En marketing, este enfoque se emplea para analizar grandes volúmenes de datos, como interacciones en redes sociales, con el fin de segmentar audiencias de manera más precisa.

¿Qué es el preprocesamiento de datos?

El preprocesamiento de datos se refiere al proceso de convertir datos en bruto en un formato más útil y comprensible para el análisis y la creación de modelos. Este paso es crucial, ya que contribuye significativamente a la calidad y eficiencia de los modelos de Machine Learning (ML) al solucionar problemas como la ausencia de datos, el ruido, las inconsistencias y los valores atípicos.

Este proceso incluye varias técnicas, como la imputación de valores faltantes, la normalización de datos y la detección y eliminación de outliers. Al realizar estas transformaciones, se busca preparar un conjunto de datos que sea más representativo y que facilite el aprendizaje del modelo, mejorando así su rendimiento y precisión.

Ventajas del preprocesamiento de datos para el aprendizaje automático

Limpieza de datos

La limpieza de datos es un paso crucial en el proceso de preprocesamiento de datos en machine learning. Consiste en detectar y corregir errores o inconsistencias en el conjunto de datos para asegurar que estos sean precisos y apropiados para el análisis o el entrenamiento de modelos.

Gestión de los valores faltantes

Los datos incompletos son un desafío común en los conjuntos de datos del mundo real y pueden perjudicar el desempeño de los modelos de machine learning. Para identificar y tratar los valores que faltan, se puede realizar lo siguiente:

  • Usar estadísticas descriptivas o gráficos para localizar las columnas o características con datos faltantes. Los valores ausentes suelen representarse como NaN o Nulos.
  • Evaluar el impacto de los valores faltantes en el análisis o modelo, considerando el porcentaje de valores ausentes en cada columna y su importancia dentro del conjunto de datos.
  • También existen técnicas más avanzadas de imputación, como la imputación por regresión, por vecinos más cercanos (k-NN) o empleando modelos de machine learning que predicen los valores ausentes a partir de otras características.

Gestión de los valores atípicos

Los valores atípicos son datos que se desvían considerablemente de otras observaciones y pueden distorsionar los análisis o modelos de machine learning.

  • Para detectarlos, se pueden utilizar gráficos como boxplots, histogramas o diagramas de dispersión, que te permiten visualizar posibles valores atípicos.
  • Calcular medidas como la media, la desviación estándar y el rango intercuartil (IQR).
  • Si los valores atípicos se deben a errores o datos incorrectos, es recomendable eliminarlos usando filtros estadísticos.
  • Aplicar transformaciones como logaritmos, raíces cuadradas o la transformación Box-Cox puede hacer que los datos sean más simétricos y reduzcan el efecto de los valores atípicos.

Gestión de duplicados

Los registros duplicados pueden sesgar los resultados inflando ciertos patrones o introduciendo sesgos en el modelo.

  • Usar funciones o librerías para identificar filas duplicadas basándose en columnas específicas o en toda la fila.
  • Si los duplicados no aportan nueva información, se pueden eliminar con librerías o métodos similares.
