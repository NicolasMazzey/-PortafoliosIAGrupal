Clasificador de colores

Objetivo

El objetivo de este proyecto es crear un clasificador de colores a partir de un dataset en el cual se indican los codigos RGB de varios colores y el nombre.


Problema

Este problema pertenece a la categoría de aprendizaje supervisado y específicamente de clasificación, ya que el objetivo es asignar un valor categórico (nombre del color) a partir de tres valores continuos (R, G, B).

El usuario debe ingresar el codigo RGB que desee y utilizando un enfoque basado en el algoritmo de clasificacion KNN, la respuesta va a ser el color "mas cercano" al ingresado.


Actividades de preparación

Verificación de valores RGB: Asegurarse de que los valores estén dentro del rango válido (0-255). Balance del dataset: Asegurarse de que haya una representación uniforme de colores básicos y tonos intermedios.

Tareas de selección de atributos

Dado que los valores RGB son directamente relevantes para el problema, no se requiere una selección de atributos compleja. Sin embargo, consideramos: Usar combinaciones de características como intensidad promedio o luminancia, pero estas fueron descartadas para mantener el modelo simple. Se utilizaron directamente los valores r, g, b como atributos para entrenar el modelo.

Identificación de posibles enfoques de algoritmos de ML

Dos posibles enfoques se consideraron para resolver el problema: Modelo basado en distancias (heurístico): Comparar los valores RGB ingresados con los del dataset y calcular la distancia euclidiana entre ellos. El color con la menor distancia es el predicho. Justificación: Es simple, interpretable y no requiere un entrenamiento explícito. Modelo de clasificación supervisada (k-Nearest Neighbors): Entrenar un modelo k-NN con el dataset, usando los valores RGB como entrada y el nombre del color como salida. Justificación: Se adapta bien a problemas basados en proximidad y puede generalizar mejor en caso de colores no presentes en el dataset.

Configuración de parámetros para cada modelo

Enfoque basado en distancias: Se calculó la distancia euclidiana. Este método no requiere parámetros adicionales. Modelo k-NN: Número de vecinos (k): Se probó con valores de k=1, k=3 y k=5. Distancia: Métrica euclidiana para evaluar similitud.
